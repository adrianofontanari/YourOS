# YourOS Vault Pages

### YourOS Vault Pages:
This section lists all your pages. I would recommend skimming this section to get some familiarity with YouroS but do not spend too much time on it. After a few weeks of usage, you can always refer back to this section.


- 🏡Home: The Home page is designed to become the dashboard of your life. Follow the steps included in this handbook to correctly configure the system.

1) Growth:
- 🪴 Today (Task Manager): This page shows the activities due today. It also includes the weekly priorities, an inbox for notes with the tag "inbox", and a scratch book
- ☘️ Upcoming (Task Manager): This page shows the activities due tomorrow and in the next 7 days
- 🌿 This Week (Task Manager): This page shows the activities planned for this week. Edit this page to update the list of tasks appearing in Today and Upcoming. New tasks added via quick capture (CMD + T) will be added here with today as the due date. It is good practice to update the list once a week (during the review) and move the completed activities to the tasks log. To collect the tasks select from the command palette: "Task Collector (TC): Mark task".
- 👨🏼‍🌾 Garden (Task Manager): This page lists unplanned activities (no deadline/deadline > 1 week). During the weekly review process move postponed tasks from [[🌿 This Week]] to this section or vice-versa: planned activities to the [[🌿 This Week]].
- 🔁 Periodic: This page lists periodic tasks. It is good practice to regularly move the completed activities to the [[Journal/TasksLog/2024|✅ Tasks Log]]. To collect the tasks select from the command palette: "Task Collector (TC): Mark task". YourOS comes with a list of pre-defined activities, add the start and due date to make them work (e.g. as for the weekly review): 🛫 2024-02-04 📅 2024-02-04
- 📚 Library: This page shows a dataview table that lists the notes in the "Books" folder
- 📕 Antilibrary: An **antilibrary** is a collection of books that are owned but have not yet been read. The term was coined by [Nassim Nicholas Taleb](https://en.wikipedia.org/wiki/Nassim_Nicholas_Taleb "Nassim Nicholas Taleb"). The concept it describes has been compared to the Japanese _[tsundoku](https://en.wikipedia.org/wiki/Tsundoku "Tsundoku")_. [Source](https://en.wikipedia.org/wiki/Antilibrary)
- 👨🏼‍🎓 Education: This page shows a dataview table that lists the notes with the  "education" tag. Education pages could refer to the title of a university major (e.g. MSc in Artificial intelligence); the sub-pages could include the courses.
- 📜 Quotebook: This page can accommodate a collection of quotes. Note: to see a quote in the daily journal use the following note: [[Quotebook|Daily Quotebook]]. Would you like to define your life principles and you do not know where to start? Ray Dalio's books can be a good starting point. Note: The quotes in the Daily Quotebook must follow specific formatting rules to be correctly displayed (follow the example included in the note to avoid issues).

2) Identity
- 🌳 Projects: This page shows the active projects in the short-term (next 12 months), mid-term (next 2-4 years), and long-term (>5 years). Completed projects will appear in the [[🏆 Museum]]. Note: YourOS treats goals as projects (to make things happen goals should be projects with a defined deadline). You can add new projects by selecting New Project from the command palette menu. To see PDF files in the folder of a specific project, add the folder name to the data view query on the project page.

#### 🌠 Project/Goal Rewards Guide

- Daily
	- **DEFINITION:** A successful project that requires a few days of work
		**EXAMPLE:** Write an article for the website
		**REWARD:** 🛴
- Weekly
	- **(1) DEFINITION:**  A successful  project that requires a few weeks of work
		- **EXAMPLE:** Write a report for the website
		- **REWARD:** 🛵
- Performance-based
	- **DEFINITION:**  Reach a goal based on a continuous commitment (it depends mostly on you) - can include tier 2,3 only, the focus is on consistency
	- **EXAMPLE:** Adherence to routines (e.g. reading a book 80% of the time)
	- **REWARD:** 🚗 
- Monthly
	- **DEFINITION:** A successful project that requires a few months of work or adherence to routines Tier 1 goals
	- **EXAMPLE:** Learn Python Fundamentals
	- **REWARD:** 🏎  
- (mostly) Performance-based
	- **DEFINITION:** Reach a goal (expected) based on performance (it depends also on external factors) - can include tier 1,2 only
	- **EXAMPLE:** Get a salary raise
	- **REWARD:** 🚁  
- Performance-based
	- **DEFINITION:**  🛩️  Reach a goal (more than what was expected) based on performance (it depends also on external factors) - can include tier 1 only
	- **EXAMPLE:** Change Job, Job promotion
	- **REWARD:** 🛩️  
- Inspirational
	- **DEFINITION:** Life-changing goals
	- **EXAMPLE:** Buy a House
	- **REWARD:** ✨

Personal Life Manifesto: A personal manifesto is a written statement outlining your aspirations and desires in life. This is also important for job search. Would you like to define your life principles and you do not know where to start? Ray Dalio's books can be a good starting point. These can be later added to the [[Quotebook]] and appear in the daily journal.
- 📔 Journal: This page shows the collection of Journals for each year. Add the tag "journal-showcase" to the year's collection page to see the notes in this data view query
- Year - Collection: This page shows the diary pages for the respective year
- 🙏 Gratitude Log: This data view query lists all the diary entries in the "Today I am grateful for..." section of the daily journal
-  🙌 Praise Log: This data view query lists all the diary entries in the "Praise Received..." section of the daily journal
- 🗂️ Memories: This data view query lists all the diary entries in the "A moment to remember" section of the daily journal
- 📘 Life Learnings: This data view query lists all the diary entries in the "Life Learning Log" section of the daily journal. You can add a keyword in the daily journal to classify the life learnings (e.g. l-growth: X, l-mindset: Y). An example of a life-learning could be: "l-health: drinking coffee after 5 pm will impact my sleep". Life learnings will also appear in the growth section of the journal reviews (e.g. weekly reviews).
- 🏆 Museum: It is always good to achieve a goal/project. You do not need to be famous to have a personal museum. This space lists completed goals/projects. You can complete a project by changing the tag from "project/active" to "project/completed" in the project's properties. You can add academic achievements (e.g. certifications) in the dedicated section. Additionally, you can add pictures of personal historical moments in the folder "Attachments/Museum". No longer interested in a project/goal? Change the tag from "project/active" to "project/stopped" to add it to the [[☁️ Projects Memorial]].
- ☁️ Projects Memorial: Not all projects/goals are meant to be completed. To close a project in YourOS change the tag "project/active" to "project/stopped" on the project page. Interrupted projects/goals will be listed on this page.
- [[🗃️ Archive]]: Following the P.A.R.A. method, this page lists [[🌻 Areas]] that have been archived. For example, did you change your job? Move the old employer's page to this section by changing the tag "area/active" to "area/archived" in the Area's properties.
- 🌻 Areas: This page shows the activity areas classified by dimension. Following the P.A.R.A. method, an Area is a zone of responsibility that can have multiple sub-areas. You can add a new area or sub-area by selecting New Area or New Sub-Area from the command palette menu. An example of an Area could be a degree program (e.g. MS in Artificial Intelligence). Sub-areas could refer to the different courses of the program (e.g. Machine Learning). Areas can be archived by changing the tag from "area/active" to "area/archived" in the Area's properties. To see the linked notes of an Area change the "placeholder" tag with the name of the area. Remember to add the same tag to the notes you would like to be linked with the area.

- 💾 Backup: A formal backup strategy allows you to never miss a file. I would recommend backing up YourOS vault in its integrity at least once a month. Much can be said about backup strategies (e.g. see the 3-2-1 backup rule), the important thing is to have a regular copy of your data in several places. For privacy-focused storage, a Network-attached storage (NAS) could be an option. However, most users may just need an external HD. Use this note to formalize your backup strategy and backup info (e.g. date of last backup). Do not forget to put it into action by creating a recurring task). You may also want to explore more advanced cybersecurity practices, such as buying a physical 2FA security key (e.g. Yubico)
Backup Logs: Example:
- Last Backup: January 10, 2024
- Additional info: Saved on iCloud + offline HD. Photos not exported
Backup Places: Example:
- iCloud (recurring subscription)
- External Hard Disk: bought on 05 Jan 2021. Average life duration: 5 years.

> [!tip] This page shows the activity areas classified by dimension. Following the P.A.R.A. method, an Area is a zone of responsibility that can have multiple sub-areas. You can add a new area or sub-area by selecting New Area or New Sub-Area from the command palette menu. An example of an Area could be a degree program (e.g. MS in Artificial Intelligence). Sub-areas could refer to the different courses of the program (e.g. Machine Learning). Areas can be archived by changing the tag from "area/active" to "area/archived" in the Area's properties. To see the linked notes of an Area change the "placeholder" tag with the name of the area. Remember to add the same tag to the notes you would like to be linked with the area.

```dataview
TABLE WITHOUT ID (link(file.path)) as Area, Dimension
FROM "Areas" and #area/active
SORT area Asc
```
